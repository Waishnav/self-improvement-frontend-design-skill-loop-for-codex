# Version 25

## Experiment Topology

vertical

## Isolation Mode

isolated-fresh-app

## Skill Baseline

custom-user-authored-skill

## Hypothesis

Expanding the art-direction planning phase and anti-pattern bans in the skill should improve concept originality and reduce repeated AI-default landing structures.

## Mutation Axis

Axis: 2 (`Section graph diversity`)

## Exact Skill Change

- Expanded Phase 1 concept scaffolding with concrete artifact mappings and explicit "wrong-fit" checks.
- Added stronger anti-default guidance in typography, color, and layout sections.
- Added more explicit banned-pattern language, but without hard quantitative depth gates.

## Expected Visual Delta

- Higher route-to-route visual identity separation.
- Better non-template composition decisions versus previous versions.
- Deeper section structures after hero.

## Measured Result

- Headless generation completed successfully via `artifacts/version-25-run-01` (attempt 1).
- Generated `t4-canvas` app includes routes `/`, `/1`, `/2`, `/3`, `/4`, `/5`.
- Screenshot capture completed for `/1.. /5` across desktop/tablet/mobile (+ `laptop.png` alias).
- Server lifecycle verified: dev server was stopped after capture and port `4000` is clear.
- Rubric score: **14.7 / 20** (average **1.47 / 2**), delta **-1.9** vs `version-24` (**16.6 / 20**).
- Outcome summary: strong stylistic distinctiveness, but structural depth regressed; most routes behave like hero-first pages with insufficient below-fold modules.

## Keep / Drop

Drop this mutation as-is for production loop progression.
